# Формат тестов

Файлы:

- `auditor.json` — вопросы для аудитора;
- `operator.json` — вопросы для оператора.

Код поддерживает текущий формат: JSON-массив с одним объектом теста, внутри которого есть поле `questions`.

```json
[
  {
    "test_title": "Тест для оператора",
    "project": "Мониторинг цен",
    "recommended_settings": {
      "questions_count": 20,
      "passing_score": 16
    },
    "questions": [
      {
        "question": "Что нужно сделать перед проверкой фото?",
        "type": "single_choice",
        "options": {
          "A": "Сразу переписать цену.",
          "B": "Прочитать название товара и комментарий.",
          "C": "Проверить только цену.",
          "D": "Открыть следующий товар."
        },
        "correct_answers": ["B"],
        "explanation": "Сначала нужно понять условия задания."
      }
    ]
  }
]
```

Также поддерживается простой формат: JSON-массив вопросов без общего объекта.

Правила:

- вариантов ответа может быть от 2 до 5;
- варианты обозначаются буквами `A`, `B`, `C`, `D`, `E`;
- для одного правильного ответа используйте `correct_answers: ["B"]`;
- для нескольких правильных ответов используйте `correct_answers: ["A", "C", "D"]`;
- поле `type` может быть `single_choice` или `multiple_choice`;
- количество вопросов берется из JSON-файла, отдельная переменная окружения для количества вопросов больше не нужна.
